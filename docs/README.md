# Skills 文档总览

`docs/` 用来存放对仓库内 skills 的详细说明，重点解释每个 skill 的定位、能力边界、适用场景和推荐工作流。

## 当前文档

- [yuque-openapi.md](./yuque-openapi.md)：详细介绍 `yuque-openapi` skill 的具体能力、作用、常见使用场景和安全注意事项。
- [feishu-doc-sync.md](./feishu-doc-sync.md)：仓库级说明 `feishu-doc-sync` 的定位、tenant/user 两种模式、当前实测能力和推荐阅读路径。
- [skill-learning-guide.md](./skill-learning-guide.md)：结合语雀和飞书两个 skill 学习如何阅读、拆解和构建新 skill。
- [skill-design-template.md](./skill-design-template.md)：从需求到 skill 设计表的实战模板（可直接复制填写）。
- [skill-quickstart.md](./skill-quickstart.md)：用最短路径快速做出一个可用的 skill。
- [skill-authoring.md](./skill-authoring.md)：讲解如何在本仓库中从 0 到 1 制作一个新的 skill。
- [skill-spec.md](./skill-spec.md)：整理 skills 的结构规范、命名规范、编写规范和维护建议。

## 阅读建议

如果你是第一次接触这个仓库，建议按下面顺序阅读：

1. 先看仓库首页 [../README.md](../README.md)，快速了解当前有哪些 skills。
2. 如果你是做语雀同步，继续看 [yuque-openapi.md](./yuque-openapi.md)。
3. 如果你是做飞书文档同步，继续看 [feishu-doc-sync.md](./feishu-doc-sync.md)。
4. 如果你想先学会怎么读、怎么仿照现有 skill 去做，先看 [skill-learning-guide.md](./skill-learning-guide.md)。
5. 如果你想先把需求翻译成结构设计，再看 [skill-design-template.md](./skill-design-template.md)。
6. 如果你想快速新建一个 skill，再看 [skill-quickstart.md](./skill-quickstart.md)。
7. 如果你想继续扩展仓库能力，再看 [skill-authoring.md](./skill-authoring.md) 和 [skill-spec.md](./skill-spec.md)。
